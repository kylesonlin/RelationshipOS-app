# AI Constraints & Operating Principles
## Mandatory Guardrails for AI-Assisted RelationshipOS Development

> **🛡️ AI Operating Principles**: These constraints are MANDATORY for all AI assistants working on RelationshipOS. They ensure revolutionary standards while preventing scope creep and maintaining strategic alignment.

---

## 🎯 **MANDATORY AI OPERATING PRINCIPLES**

### **1. Single Responsibility Principle**
```markdown
Each AI session must focus on ONE clear objective:
✅ ALLOWED: "Implement Oracle Engine search interface"
✅ ALLOWED: "Optimize relationship scoring algorithm"
❌ FORBIDDEN: "Build Oracle Engine and also add social features"
❌ FORBIDDEN: "Fix bugs and add new features and refactor architecture"

Rationale: Single focus ensures quality and prevents context dilution.
```

### **2. Revolutionary Standards Validation**
```markdown
EVERY suggestion must pass the 5-test framework:
1. Oracle Test: Does this make Oracle Engine more magical?
2. Platform Test: Does this serve the RelationshipOS ecosystem?
3. 10x Test: Is this 10x better than alternatives, not 10% better?
4. Magic Test: Will users say "this feels like the future"?
5. Focus Test: Does this align with relationship intelligence mission?

❌ NEVER suggest features that fail ANY of these tests.
```

### **3. Backwards Compatibility Commitment**
```markdown
AI assistants MUST maintain backwards compatibility:
✅ Enhance existing features without breaking functionality
✅ Add new capabilities while preserving current workflows
✅ Improve performance without changing user experience
❌ Never break existing Oracle Engine functionality
❌ Never remove features without explicit strategic approval

Exception: Only with @exec (full Virtual Executive Team) approval.
```

### **4. Performance First Architecture**
```markdown
Oracle Engine <10 second response time is NON-NEGOTIABLE:
✅ Always consider performance impact of suggestions
✅ Recommend caching strategies for expensive operations
✅ Optimize database queries and API calls
✅ Use background processing for non-critical tasks
❌ Never suggest features that could slow Oracle responses
❌ Never recommend synchronous operations that block user experience

Every code suggestion must maintain or improve performance.
```

### **5. Documentation Currency**
```markdown
AI assistants MUST maintain documentation integrity:
✅ Update docs/CURRENT_STATUS.md with every significant change
✅ Verify AI_CONTEXT.md accuracy after modifications
✅ Update cross-references when changing document structure
✅ Document decisions and reasoning for future AI context
❌ Never make changes without updating relevant documentation
❌ Never leave broken links or outdated references

Documentation is code. Treat it with the same rigor.
```

## ❌ **FORBIDDEN AI ACTIONS**

```markdown
❌ Project management (tasks, deadlines, collaboration tools)
❌ Generic AI features (chatbots, content generation, general AI)
```

```markdown
❌ Recommend shortcuts that compromise quality gates

Process integrity ensures long-term success and team coordination.
```
