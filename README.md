## 🏗️ **Technical Architecture**

### **Technology Stack**:
- **Frontend**: Next.js 14 (App Router) + Radix UI + Tailwind CSS
- **Backend**: Vercel Edge Functions + Supabase (PostgreSQL)
- **AI Engine**: OpenAI GPT-4 Turbo + Pinecone (Vector Database)
- **Deployment**: Vercel (app.wisedom.ai)
- **Integrations**: LinkedIn API, Gmail API, Calendar APIs

### **Database Schema**:
Relationship-centric design optimized for B2B SaaS:
- `people` - Contact and relationship data
- `organizations` - Company and institutional data  
- `relationships` - Connection mapping and intelligence
- `queries` - Oracle Engine search history
- `intelligence_cache` - AI-generated insights and analysis

## 🛡️ **Decision Framework**

**Before suggesting ANY feature or change, apply our 5-test framework**:

1. **Oracle Test**: Does this make Oracle Engine more magical?
2. **Platform Test**: Does this serve the RelationshipOS ecosystem?
3. **10x Test**: Is this 10x better than alternatives, not 10% better?
4. **Magic Test**: Will users say "this feels like the future"?
5. **Focus Test**: Does this align with relationship intelligence mission?

### **Forbidden Features** (Anti-Scope Creep):
❌ Traditional CRM features (pipelines, deal tracking)  
❌ Social network features (posts, feeds)  
❌ Communication tools (chat, email, video)  
❌ Project management (tasks, deadlines)  
❌ Generic AI features (chatbots, content generation)
